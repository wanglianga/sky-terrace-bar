# Sky Terrace 38 · 城市露台酒吧座位与日落时段页面

基于 Vue3 + Vite 构建的城市高空露台酒吧预订系统，包含顾客手机端和店员平板端双视角。

---

## 原始需求

> 请制作城市露台酒吧座位与日落时段页面，Vue3 页面围绕楼层入口、露台分区、天际线朝向、落日晚霞、驻唱安排、低消套餐、雨棚开合和座位余量展开。顾客在手机上挑选靠栏杆、高脚桌、沙发角或吧台，能同时看到视野照片、可坐人数、到店窗口、生日布置和夜景氛围；店员平板端需要处理迟到保留、临时拼桌、室内迁移和包厢改价。页面从黄昏到夜间要有连续的视觉变化，座位图不能只好看，还要让价格差异、遮挡风险、风力影响和订金规则在一次浏览里看明白。

---

## 项目简介

本项目模拟了 38 层高空露台酒吧的完整预订场景：

- **顾客手机端** `/customer`：分区浏览、座位筛选、时段选择、视野预览、套餐选购、生日布置、实时余量
- **店员平板端** `/staff`：迟到保留、临时拼桌、室内迁移、包厢改价、桌台状态看板、快速操作
- **动态视觉**：天空颜色、太阳/月亮、星空、城市灯光、环境光随选择时段（下午茶→日落→蓝调→夜间→深夜）连续过渡变化

## 技术栈

- Vue 3.4 + Composition API
- Vite 5 构建工具
- Vue Router 4（Hash 路由）
- 纯 CSS 动态视觉动画，无需额外 UI 库

## 启动方式

### 前置要求

- **Node.js** ≥ 18.x
- **npm** ≥ 9.x 或 **pnpm** / **yarn**
- 如需 Docker 启动：**Docker** ≥ 20.x 与 **Docker Compose** ≥ v2

---

### 方式一：Docker 一键启动（推荐）

#### 1. 构建并启动

```bash
docker compose up --build
```

如需后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问地址

顾客端： http://localhost:8080/#/customer
店员端： http://localhost:8080/#/staff
（首页会自动跳转到顾客端）

#### 3. 停止与清理

```bash
docker compose down
```

> Docker 方式已包含 nginx 静态托管，gzip 压缩，前端资源缓存。
> 构建镜像体积约 150MB（基于 nginx:alpine）。

---

### 方式二：本地开发模式

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发服务器

```bash
npm run dev
```

访问地址： http://localhost:5173

- 顾客端： http://localhost:5173/#/customer
- 店员端： http://localhost:5173/#/staff

#### 3. 生产构建与预览

```bash
npm run build
npm run preview
```

预览地址： http://localhost:4173

## 功能一览

### 顾客端（`#/customer`）

| 模块 | 说明 |
|------|------|
| 楼层入口信息 | 38 层电梯厅、礼宾引导、营业时段、着装要求 |
| 露台分区 | 落日主区 / 天际线区 / 江景区 / 吧台休闲区 / 室内包厢区 |
| 天际线朝向 | 正西/正北/正南/正东 视野指数、落日评分、风力等级 |
| 落日晚霞 | 时段切换驱动天空渐变、太阳轨迹、星空、城市灯光变化 |
| 驻唱安排 | 爵士弹唱 / DJ / 乐队演出时间线、所在分区、是否正在上演 |
| 低消套餐 | 4 档套餐（¥588 - ¥5888），含情侣 / 聚会 / 商务 / VIP 标签 |
| 雨棚开合 | 一键切换雨棚状态，视觉提示当前覆盖 |
| 座位余量 | 5 个分区实时空位、总容量条、颜色区分价格带 |
| 座位挑选 | 靠栏杆 / 高脚桌 / 沙发 / 吧台 / 方桌 / VIP 六种类型图标筛选 |
| 视野照片 | 每个座位弹窗展示该分区真实感渐变视野图 + 视野标签 |
| 可坐人数 | 按人数（2/4/6/8+）快速过滤座位 |
| 到店窗口 | 5 档时段卡片，标注价差倍率、热门标记、天空颜色索引条 |
| 生日布置 | 基础（¥99）/ 尊享（¥399）布置选项叠加 |
| 夜景氛围 | 星空灯、背景音乐、调酒火焰秀、烛光的分区差异展示 |
| 价格差异 | 不同座位图块颜色映射、低消直接显示在座位上 |
| 遮挡风险 | 有遮挡座位标注 ⚠️、边框显示为双线 |
| 风力影响 | 高风区标注 💨、分区卡片展示风力等级 |
| 订金规则 | 30% 订金、取消规则、到店可抵扣说明卡片 |
| 底部结算栏 | 选中总价 + 订金实时汇总、一键支付按钮 |

### 店员端（`#/staff`）

| 模块 | 说明 |
|------|------|
| 顶部统计卡 | 空位 / 保留 / 预订 / 入座 / 总数 5 个维度实时看板 |
| 迟到保留 | 选择座位 → 设定 5-60 分钟保留时长 → 填写客户信息 → 生效并显示倒计时 |
| 临时拼桌 | 选中 ≥2 个座位 → 预览合并容量 → 一键拼桌并标记「拼桌」徽章 |
| 室内迁移 | 选中室外座位 → 自动分配到室内包厢区 → 支持一键还原 |
| 包厢改价 | 选中座位 → 覆盖原价 / 快速价 / 一键重置为基准价 |
| 分区切换 | 5 个分区独立座位图，带数量对比 |
| 图例说明 | 8 种状态/风险颜色图例 |
| 当前桌台面板 | 30 个桌台卡片化列表：状态标签、预订人、保留倒计时、到店时长、速操作 |
| 速操作按钮 | 速留 / 释放 / 还原 直接在卡片上操作 |

## 注意事项

1. 本项目为前端演示，所有座位、预订、天气、驻唱数据均为 `src/data/seatsData.js` 内的模拟数据，刷新后重置。
2. 顾客端顶部导航可切换至店员端（或直接访问路由 `#/staff`）。
3. 天空与灯光视觉受 **时段选择条** 控制，切换不同时段可观察连续色彩过渡、太阳位置、星星出现、城市灯光亮起等效果。
4. 页面布局对移动端做了适配，建议使用手机模拟器或真实设备访问顾客端以获得最佳体验；店员端适合 10"+ 平板屏幕。
5. Docker 构建需要可访问的 npm 镜像源，国内环境如构建慢可在 Dockerfile 切换为 taobao 镜像。
